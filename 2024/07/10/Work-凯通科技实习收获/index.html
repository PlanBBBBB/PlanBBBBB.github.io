记录第二份实习的实习收获

项目

EasyData(ETL同步工具）

文件扫描工具支持：本地扫描、FTP扫描和SFTP扫描三种扫描方式。通过配置可以定时启动扫描。

SQL LOAD

https://blog.csdn.net/Lqq77s/article/details/89403199

具体流程

  1. 文件扫描工具先扫描文件到数据库里进行记录
  2. 数据分析工具通过待完成标识和任务标识去定时处理这些任务
  3. 对于每个新加入的任务，需要在数据库先进行配置任务，任务所需要执行的类，这个类是可以动态扩展的
  4. 需要在代码写入的其实就是一个包含新的处理逻辑的类，这个类去实现指定好的接口后，自定义实现处理逻辑，比如我实现的就是从excel表中读取每行数据交给计划任务（当然不止可以从excel表里读数据，还可以从数据库里从ftp文件，csv文件里读取）
  5. 交给计划任务之后，任务会执行sqlload的操作，去批量操作数据库提高效率

全量同步原理

全量同步只是从数据源S1读取数据，进行一些基本处理后，便将数据导入目的库

增量同步原理

增量同步的处理过程：

  • 从FTP文件或数据库导出数据到文件。
  • 对数据进行取模分区，这样可将数据进行分类，主键相同的数据一定在同个分区中。
  • 将每个分区按一定数量切割分片并排序，为后面的多路归并做准备。
  • 最后按分区对分片进行多路归并求差异，将S1和S2相同分区的分片文件进行多路归并求差异。

大文件分区分片的过程中，都是按行读取文件，读取一定数量进入内存中然后写入小文件，重复这个过程，直到大文件处理完成，这整个流程使得能够处理大文件，同时避免内存溢出。

两份数据求差异，最终生成三个文件：add.txt，edit.txt和del.txt。数据以S1为准，以主键作为数据唯一标识。这三份文件含义如下：

add.txt：S1有某个主键，但S2没有，则该条数据为新增数据。

edit.txt：S1和S2都有该主键，但其它字段不同，则该条数据需要更新。

del.txt：S1没有该主键，但S2有，则该条数据需要删除。

多路归并原理

一个分区中的每个分片内部的数据都是有序的，但分片之间并不是有序的。在一次求差异过程中，每个分片，都只取前m条数据，如果S1一个分区中有n个分片，则一共取s=n*m条数据；将这s条数据进行从小到大排序；那么，这s条数据中的前m条数据，就是这整个分区的最小的m条数据。

封装EasyExcel

概述

通过策略模式封装 EasyExcel 实现导入功能，加入心跳检测提升导入的准确性，优化资源利用

封装了一个名为ExcelImportThread的类，该类实现了Runnable接口，意味着它可以作为一个线程运行。这个类主要用于处理Excel文件的导入流程，包括解析Excel文件、数据适配、数据保存到数据库等步骤，并且具备一些额外的功能，如心跳检测和文件清理。

主要功能描述

  1. 线程管理和状态追踪:
    1. 使用HashMap来存储每个导入任务的ExcelImportThread实例，以便跟踪任务的状态。
    2. 定义了导入任务的不同状态，如解析中、数据适配中、入库中、完成、异常、取消等。
  2. 心跳检测:
    1. 每个ExcelImportThread实例都有一个lastQueryTime字段，记录了上次查询的时间点。
    2. 在checkImportTimeOut方法中，通过比较当前时间和lastQueryTime的时间差来判断是否超过了心跳监测的最大时间间隔MAX_TIME_OUT（默认为10秒）。
    3. 如果超时，则调用cancelImport方法取消导入任务，并从threadMap中移除该任务。
  3. 文件管理:
    1. 文件会保存在一个临时目录下，并且有最长保存时间限制MAX_FILE_TIME_OUT（默认为1小时）。
    2. deleteTimeOutFile方法用于检查并删除超过保存时间限制的文件。
  4. 导入流程:
    1. 解析Excel文件，将数据存储在内存中。
    2. 对数据进行适配处理。
    3. 将适配后的数据保存到数据库。
    4. 如果有数据适配失败的情况，会记录失败原因并写回原Excel文件。
  5. 错误处理:
    1. 在每个关键步骤中都加入了异常捕获逻辑，以确保程序的健壮性。
    2. 如果发生异常，会设置导入状态为异常，并记录错误信息。

ExcelImportThread类通过几个关键方法来根据实例的状态进行不同的操作。下面将详细解释这些方法以及它们是如何根据实例的状态来执行不同操作的。

实例状态及操作

初始化状态:

  1. 当ExcelImportThread实例创建时，lastQueryTime会被设置为当前时间。
  2. 实例的状态默认为未开始处理，即修改importState，表示“正在解析”。

处理过程:

  1. 随着处理的进行，importState会被更新以反映当前的状态，例如：
    • 正在适配数据
    • 正在入库
    • 导入完成
    • 导入异常
    • 取消导入

状态查询:

  • 当外部需要查询ExcelImportThread实例的状态时，可以通过调用getImportStateByUUId方法来获取状态信息。

  • 这个方法首先会检查该实例是否存在，并更新lastQueryTime，然后返回当前的状态信息。

心跳检测:

  • 心跳检测是通过checkImportTimeOut方法来实现的，该方法会检查当前时间与lastQueryTime之间的时间差是否超过了最大时间间隔MAX_TIME_OUT（默认为10秒）。

  • 如果超过了这个时间间隔，那么就会认为该线程已经超时，从而调用cancelImport方法取消导入任务。

定时任务:

  • 可以通过设置一个内部的定时任务来定期检查所有ExcelImportThread实例的心跳。

  • 这个定时任务会定期调用checkAllImportTimeOut方法，该方法遍历threadMap中的所有实例，并调用每个实例的checkImportTimeOut方法来进行心跳检测。

总结

ExcelImportThread类通过维护实例的状态（importState）和最近一次查询的时间（lastQueryTime）来根据实例的状态进行不同的操作。外部可以通过查询方法来获取实例的状态，并更新查询时间；而心跳检测则通过定时任务来定期执行，以确保长时间没有活动的实例能够被及时发现并取消。
